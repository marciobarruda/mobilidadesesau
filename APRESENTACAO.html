1. Apresentação

O Portal de Mobilidade Interna da SESAU é uma solução digital desenvolvida para modernizar, simplificar e dar transparência ao processo de movimentação interna dos servidores da Secretaria de Saúde do Recife.

O sistema substitui fluxos manuais por um ambiente web intuitivo onde o servidor pode:

  • Consultar seus dados funcionais.
  • Solicitar mobilidade (transferência) para outras unidades.
  • Acompanhar status e obter comprovantes.

2. Público Alvo

  • Servidores Ativos da SESAU: Todos os vínculos (Estatutários, CTD, Comissionados).
  • Equipe de RH / Gestão de Pessoas: Recebimento de dados padronizados.

3. Guia de Usabilidade (Continuação)

Fluxo Passo a Passo

  1. Início: Clique em "Nova Solicitação" no dashboard.
  2. Identificação: Conferência automática de dados funcionais (Matrícula, Nome, Cargo).
  3. Contatos: Atualização obrigatória de WhatsApp e E-mail para notificações.
  4. Preferências: Seleção de Unidades de Destino (múltipla escolha) e opção de Permuta.
  5. Justificativa: Informar motivo e anexar documentos (se necessário).
  6. Conclusão: Envio da solicitação e recebimento do comprovante PDF.

Acompanhamento

O status ("Em análise", "Aprovado", "Aguardando Vaga") pode ser consultado a qualquer momento no painel "Minhas Solicitações".

4. Dados Técnicos

Arquitetura Frontend

  • HTML5 / CSS3 / ES6+
  • Bootstrap 5.3: Layout responsivo.
  • Choices.js: Componente de seleção avançada.
  • Google Fonts: Inter e Public Sans.
  • Estilização: CSS Variables (Azul SESAU).

Banco de Dados (PostgreSQL)

  • mobilidade_segtes_relacao_nominal: Dados mestres de pessoal.
  • unidades_sesau: Catálogo de unidades/turnos.
  • ranking_mobilidade: Tabela de processamento de filas.

Integração Backend (n8n)

Orquestração completa via Webhooks e Nodes:

  • Webhooks: Recebimento de formulários e renderização HTML.
  • Nodes Code (JS): Validação e sanitização de dados.
  • PostgreSQL Nodes: Persistência e consultas.
  • SOAP Request: Integração com SE Suite.
  • Mailgun/Gmail: Disparo de notificações.

4. Dados Técnicos (Fluxo de Processamento)

Detalhes da lógica implementada no workflow n8n:

  1. Recepção e Validação: Webhook recebe POST. Scripts JS formatam datas (DD/MM/YYYY → YYYY-MM-DD) e padronizam textos (Upper/Lowercase).
  2. Persistência:
    • Atualiza contatos na tabela mestre dos servidores.
    • Remove solicitações antigas pendentes (limpeza de histórico).
    • Insere novo registro em mobilidade_dados_recebidos.
  3. Ranking Automático: Execução de Stored Procedure que classifica candidatos baseado em Prioridade, Data Solicitação, Admissão e Nascimento.
  4. Integração SE Suite (BPM): Chamada SOAP (newChildEntityRecord) para criar processo administrativo oficial automaticamente.
  5. Notificação: Envio de e-mails transacionais com protocolo gerado.
